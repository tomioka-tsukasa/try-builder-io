# Builder.io デスクトップアプリ操作ガイド

> 作成日：2025年10月16日
> 対象：Builder.ioデスクトップアプリケーションの具体的な操作方法と機能

## 1. Builder.ioデスクトップアプリ概要

Builder.ioは、WebCatalog Desktopを通じてMac、Windowsでデスクトップアプリとして利用できます。ブラウザ版と同様の機能に加え、デスクトップ最適化された操作体験を提供します。

### 1.1 主要な特徴
- **ネイティブアプリ体験**: ブラウザタブに埋もれない独立したアプリ環境
- **ファイルシステム統合**: ローカルファイルとの連携
- **通知機能**: デスクトップ通知によるワークフロー管理
- **マルチウィンドウ対応**: 複数プロジェクトの同時作業

## 2. メインナビゲーション：Design / Interact / Code

Builder.ioのプロジェクト編集画面では、3つの主要モードを切り替えながら作業を進めます。

### 2.1 Design Mode（デザインモード）

**概要**: ビジュアルエディターによる直感的なコンテンツ作成

#### 主要機能
- **ドラッグ&ドロップエディター**
  - コンポーネントライブラリからの要素配置
  - レイアウトの視覚的な調整
  - スタイルプロパティのリアルタイム編集

- **コンポーネント管理**
  - 既存コンポーネントの再利用
  - カスタムコンポーネントの作成
  - デザインシステムとの連携

- **レスポンシブデザイン**
  - デバイス別プレビュー機能
  - ブレークポイント設定
  - レスポンシブ調整のビジュアル化

#### 具体的な操作
1. **要素の追加**: 左サイドバーからコンポーネントをドラッグ
2. **スタイル編集**: 右サイドバーでCSS properties調整
3. **レイアウト調整**: Canvas上で直接リサイズ・移動
4. **テキスト編集**: インライン編集による即座の反映

#### 対象ユーザー
- デザイナー
- マーケター
- 非技術者でコンテンツ作成を行うスタッフ

### 2.2 Interact Mode（インタラクトモード）

**概要**: 実際のユーザー体験のテストとプレビュー

#### 主要機能
- **リアルタイムプレビュー**
  - エンドユーザー視点での表示確認
  - インタラクション動作のテスト
  - フォーム入力の動作確認

- **デバイステスト**
  - モバイル、タブレット、デスクトップでの表示確認
  - タッチ操作のシミュレーション
  - 画面回転対応の確認

- **パフォーマンス確認**
  - ページ読み込み速度の測定
  - インタラクションの応答性チェック
  - 最適化が必要な要素の特定

#### 具体的な操作
1. **プレビュー実行**: 画面右上の「Preview」ボタンクリック
2. **デバイス切り替え**: 上部ツールバーからデバイス選択
3. **インタラクションテスト**: 実際にボタンクリック、フォーム入力
4. **パフォーマンス測定**: 開発者ツールとの連携

#### 対象ユーザー
- プロダクトマネージャー
- QAエンジニア
- デザイナー（UX検証時）

### 2.3 Code Mode（コードモード）

**概要**: 開発者向けの高度なカスタマイズとコード編集

#### 主要機能
- **フルソースコード編集**
  - シンタックスハイライト付きエディター
  - 自動補完機能
  - リアルタイムエラー検出

- **ファイル構造管理**
  - プロジェクト全体のファイル構造表示
  - 複数ファイルの同時編集
  - ファイル間の依存関係確認

- **開発環境統合**
  - ターミナル出力の監視
  - 開発サーバーのステータス確認
  - ホットリロード機能

- **バージョン管理**
  - GitHub統合
  - プルリクエスト作成
  - ブランチ管理

#### 具体的な操作
1. **ファイル編集**: 左のファイルエクスプローラーから編集対象選択
2. **コード記述**: 中央エディターでの直接編集
3. **エラー確認**: 下部パネルでエラーメッセージ確認
4. **プルリクエスト**: 「Send PR」ボタンでGitHub連携

#### 対象ユーザー
- フロントエンドエンジニア
- フルスタックエンジニア
- 技術的なカスタマイズが必要なケース

## 3. モード間の連携とワークフロー

### 3.1 シームレスな同期機能
Builder.ioの3つのモードは完全に同期されており、どのモードで変更を加えても他のモードに即座に反映されます。

#### 同期の仕組み
- **Design → Code**: ビジュアル編集が自動的にコードに変換
- **Code → Design**: コード変更がビジュアルエディターに反映
- **リアルタイム更新**: すべての変更が瞬時に他モードで確認可能

### 3.2 典型的なワークフロー

#### ケース1: マーケティングキャンペーンページ作成
1. **Design Mode**: マーケターがランディングページをビジュアル作成
2. **Interact Mode**: 動作確認とA/Bテスト設定
3. **Code Mode**: エンジニアが最終的な最適化とデプロイ

#### ケース2: プロダクト機能のプロトタイピング
1. **Code Mode**: エンジニアが基本的な構造とロジックを実装
2. **Design Mode**: デザイナーがUIを調整
3. **Interact Mode**: プロダクトマネージャーがUX確認

#### ケース3: 既存サイトの改修
1. **Code Mode**: 既存コードの分析と理解
2. **Design Mode**: 新しいデザインの適用
3. **Interact Mode**: 改修結果の動作確認

## 4. デスクトップアプリ固有の機能

### 4.1 ファイルシステム統合
- **ローカルアセット**: デスクトップからファイルの直接ドラッグ&ドロップ
- **プロジェクト管理**: ローカルディスクとの同期
- **オフライン作業**: 一部機能のオフライン対応

### 4.2 開発ツール統合
- **ターミナルアクセス**: システムターミナルとの連携
- **エディター統合**: VS Code等の外部エディターとの協調
- **デバッグ機能**: ブラウザ開発者ツールとの連携

### 4.3 通知とアラート
- **ビルド完了通知**: バックグラウンドビルドの完了アラート
- **エラー通知**: コンパイルエラーの即座な通知
- **共同作業通知**: チームメンバーの変更通知

## 5. 使い分けガイドライン

### 5.1 どのモードを使うべきか

| 作業内容 | 推奨モード | 理由 |
|----------|------------|------|
| 初期デザイン作成 | Design | 直感的な操作でアイデアを形に |
| コンテンツ更新 | Design | 非技術者でも安全に変更可能 |
| UX/UI検証 | Interact | 実際のユーザー体験を確認 |
| パフォーマンステスト | Interact | 実環境に近い条件でのテスト |
| 高度なカスタマイズ | Code | 技術的な制約なしでの実装 |
| デバッグ・最適化 | Code | 詳細な制御と問題解決 |

### 5.2 チーム構成別の活用方法

#### 小規模チーム（1-3人）
- **全モード活用**: 一人で複数モードを使い分け
- **Design中心**: 迅速なプロトタイピングを重視
- **Code補完**: 必要時のみ技術的調整

#### 中規模チーム（4-10人）
- **役割分担**: デザイナー（Design）、エンジニア（Code）、PM（Interact）
- **連携重視**: モード間での効率的な引き継ぎ
- **品質管理**: Interactモードでの十分な検証

#### 大規模チーム（10人以上）
- **専門特化**: 各モードの専門担当者配置
- **ワークフロー標準化**: チーム内での作業手順統一
- **レビュープロセス**: 各モードでの段階的品質チェック

## 6. パフォーマンスと制限事項

### 6.1 推奨動作環境
- **OS**: macOS 10.14以上、Windows 10以上
- **メモリ**: 8GB以上（16GB推奨）
- **ストレージ**: 10GB以上の空き容量
- **ネットワーク**: 常時インターネット接続（一部機能のため）

### 6.2 制限事項
- **同時編集**: 複数人での同一ファイル編集時の競合
- **大規模プロジェクト**: ファイル数が多い場合のパフォーマンス低下
- **オフライン機能**: 一部クラウド連携機能の制限

### 6.3 最適化のヒント
- **定期的な同期**: 変更の定期的な保存と同期
- **プロジェクト分割**: 大規模プロジェクトの適切な分割
- **キャッシュ管理**: 定期的なキャッシュクリア

## 7. トラブルシューティング

### 7.1 よくある問題と解決方法

#### モード切り替えが反映されない
- **原因**: 同期処理の遅延
- **解決策**: ページリフレッシュまたはアプリ再起動

#### コード変更がDesignモードに反映されない
- **原因**: 構文エラーまたは非対応の変更
- **解決策**: Code mode下部のエラーログ確認

#### プレビューが正しく表示されない
- **原因**: 依存関係の不足またはビルドエラー
- **解決策**: ターミナル出力の確認とパッケージの再インストール

## 8. まとめ

Builder.ioのデスクトップアプリケーションは、Design / Interact / Code の3つのモードを通じて、技術者・非技術者を問わず効率的なWeb開発を可能にします。

### 8.1 主要な価値
- **学習コストの低減**: 各ロールに適したインターフェース
- **コラボレーション促進**: モード間のシームレスな連携
- **開発効率向上**: ビジュアル編集とコード編集の自由な切り替え

### 8.2 推奨用途
- **迅速なプロトタイピング**が必要なプロジェクト
- **デザイナーとエンジニアの密接な協業**が求められる開発
- **マーケティング主導**のコンテンツ更新が頻繁な案件

Builder.ioデスクトップアプリは、現代のクロスファンクショナルチームに最適化された統合開発環境として、Web開発のワークフローを大幅に改善する可能性を持っています。
